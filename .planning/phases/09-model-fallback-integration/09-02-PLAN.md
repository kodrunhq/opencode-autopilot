  <behavior>
    Session lifecycle:
    - initSession creates fallback state for a session with the given model
    - getSessionState returns FallbackState for known session, undefined for unknown
    - cleanupSession removes all tracking state for a session (state, locks, timers, timestamps)
    - cleanupSession is safe to call on unknown sessions (no-op)

    Concurrency guards (Pitfall 1 - dual event handler race):
    - acquireRetryLock returns true on first call for a session, false on second call
    - releaseRetryLock allows subsequent acquireRetryLock to succeed
    - isDispatchInFlight returns true while retry lock is held

    Self-abort suppression (Pitfall 2):
    - recordSelfAbort stores timestamp for session
    - isSelfAbortError returns true when MessageAbortedError arrives within 2000ms of recordSelfAbort
    - isSelfAbortError returns false when MessageAbortedError arrives after 2000ms
    - isSelfAbortError returns false when no self-abort was recorded

    Stale error suppression (Pitfall 5):
    - isStaleError returns true when error model is in failedModels and differs from currentModel
    - isStaleError returns false when error model matches currentModel

    TTFT timeout:
    - startTtftTimeout creates a timer that calls the provided callback after timeoutSeconds
    - recordFirstToken cancels the TTFT timer
    - cleanupSession cancels any pending TTFT timer

    Fallback orchestration:
    - handleError with retryable error on a session with fallback chain returns FallbackPlan
    - handleError with non-retryable error returns null
    - handleError when retry lock already held returns null (Pitfall 1)
    - handleError when error is self-abort returns null (Pitfall 2)
    - handleError when error model is stale returns null (Pitfall 5)
    - commitAndUpdateState applies plan to session state immutably
    - tryRecoverToOriginal resets session to primary model when cooldown expired
  </behavior>
  <action>
    Create `src/orchestrator/fallback/fallback-manager.ts` with a class that owns per-session state. The class accepts FallbackConfig and a fallback chain resolver callback in its constructor.

    **Constructor:**
    ```typescript
    export interface FallbackManagerOptions {
      readonly config: FallbackConfig;
      readonly resolveFallbackChain: (sessionID: string, agentName?: string) => readonly string[];
    }
    ```

    **Internal per-session tracking (all private Maps/Sets):**
    - `sessionStates: Map<string, FallbackState>` -- per-session fallback state
    - `sessionRetryInFlight: Set<string>` -- lock: only one handler plans+dispatches per session (Pitfall 1)
    - `sessionAwaitingFallbackResult: Set<string>` -- flag: replay dispatched, awaiting result (Pitfall 3)
    - `sessionSelfAbortTimestamp: Map<string, number>` -- suppress self-inflicted abort errors (Pitfall 2)
    - `sessionFirstTokenReceived: Map<string, boolean>` -- TTFT tracking
    - `sessionFallbackTimeouts: Map<string, ReturnType<typeof setTimeout>>` -- TTFT timeout timers
    - `sessionParentID: Map<string, string | null>` -- parent-child mapping for subagent sync
    - `sessionCompactionInFlight: Set<string>` -- suppress stale errors during compaction

    SELF_ABORT_WINDOW_MS = 2000 (constant).

    **Public methods:**

    1. `initSession(sessionID: string, model: string, parentID?: string | null): void`
       - Creates fallback state via `createFallbackState(model)`
       - Stores in sessionStates
       - Records parentID if provided

    2. `getSessionState(sessionID: string): FallbackState | undefined`
       - Returns from sessionStates

    3. `cleanupSession(sessionID: string): void`
       - Deletes from all Maps/Sets
       - Clears TTFT timeout if present via `clearTimeout`

    4. `acquireRetryLock(sessionID: string): boolean`
       - If `sessionRetryInFlight.has(sessionID)` return false
       - Add to set, return true

    5. `releaseRetryLock(sessionID: string): void`
       - Delete from sessionRetryInFlight

    6. `isDispatchInFlight(sessionID: string): boolean`
       - Returns `sessionRetryInFlight.has(sessionID) || sessionAwaitingFallbackResult.has(sessionID)`

    7. `markAwaitingResult(sessionID: string): void` / `clearAwaitingResult(sessionID: string): void`
       - Add/remove from sessionAwaitingFallbackResult

    8. `recordSelfAbort(sessionID: string): void`
       - `sessionSelfAbortTimestamp.set(sessionID, Date.now())`

    9. `isSelfAbortError(sessionID: string): boolean`
       - Get timestamp from map. If not present return false.
       - If `Date.now() - timestamp < SELF_ABORT_WINDOW_MS` return true, else delete entry and return false.

    10. `isStaleError(sessionID: string, errorModel?: string): boolean`
        - Get state. If no state or no errorModel return false.
        - If errorModel !== state.currentModel AND state.failedModels has errorModel, return true.

    11. `startTtftTimeout(sessionID: string, onTimeout: () => void): void`
        - Clear existing timeout if any
        - Set new timeout for `config.timeoutSeconds * 1000`
        - Store timer ref in sessionFallbackTimeouts
        - Set `sessionFirstTokenReceived.set(sessionID, false)`

    12. `recordFirstToken(sessionID: string): void`
        - If `sessionFirstTokenReceived.get(sessionID) === false`:
          - `sessionFirstTokenReceived.set(sessionID, true)`
          - Clear and delete timeout from sessionFallbackTimeouts

    13. `handleError(sessionID: string, error: unknown, errorModel?: string): FallbackPlan | null`
        - If `isSelfAbortError(sessionID)` return null
        - If `isStaleError(sessionID, errorModel)` return null
        - If `!isRetryableError(error, config.retryOnErrors, config.retryableErrorPatterns)` return null
        - If `!acquireRetryLock(sessionID)` return null
        - Get state. If no state, release lock, return null.
        - Get fallback chain via `resolveFallbackChain(sessionID)`
        - Call `planFallback(state, chain, config.maxFallbackAttempts, config.cooldownSeconds * 1000)`
        - If not success, release lock, return null.
        - Return plan (caller commits after successful dispatch)

    14. `commitAndUpdateState(sessionID: string, plan: FallbackPlan): boolean`
        - Get state. If no state return false.
        - Call `commitFallback(state, plan)`
        - If committed, update sessionStates with new state. Return committed.

    15. `tryRecoverToOriginal(sessionID: string): boolean`
        - Get state. If no state return false.
        - Call `recoverToOriginal(state, config.cooldownSeconds * 1000)`
        - If result is non-null, update sessionStates. Return true/false.

    16. `markCompactionInFlight(sessionID: string): void` / `clearCompactionInFlight(sessionID: string): void` / `isCompactionInFlight(sessionID: string): boolean`
        - Add/remove/check sessionCompactionInFlight set

    17. `getParentID(sessionID: string): string | null | undefined`
        - Return from sessionParentID map

    **Tests:** Write tests using fake timers (`bun:test` supports `jest.useFakeTimers()` -- check if bun:test has timer mocking. If not, use small timeout values and Date.now() stubs). For TTFT timeout tests, use a callback spy. For handleError, inject a `resolveFallbackChain` that returns a fixed chain like `["openai/gpt-4", "anthropic/claude-sonnet-4-5"]`.

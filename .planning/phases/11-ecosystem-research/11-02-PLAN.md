**Areas to explore for novel opportunities:**
1. **Composability** -- Can we make skills/agents/commands that compose in ways competitors don't? (e.g., skills that enhance agents, agents that learn from skills)
2. **Zero-config intelligence** -- Features that work without any user configuration (auto-detect project type, auto-select relevant skills, auto-tune memory)
3. **Cross-session improvement loops** -- Beyond raw memory: pattern extraction, skill evolution, preference learning
4. **Developer workflow integration** -- PR comment review/response (per D-10), CI failure diagnosis, dependency update assistance
5. **Meta-tooling** -- Self-diagnostic, self-improving plugin capabilities (audit installed assets, validate configuration, measure impact)
6. **Context efficiency** -- Aggressive token optimization that makes the ENTIRE plugin footprint lighter, not just memory
7. **Emergent patterns** from the broader ecosystem scan that no single competitor has fully realized

**Constraints on novel opportunities (from CLAUDE.md):**
- Must work in Bun runtime
- Must be model-agnostic (no specific model dependencies)
- Must write to ~/.config/opencode/ (global target)
- Must not require external services or databases beyond bun:sqlite
- Must not conflict with OpenCode's built-in features

**Quality bar (per D-02):** Each opportunity must be realistic, implementable, and clearly valuable -- not aspirational hand-waving.

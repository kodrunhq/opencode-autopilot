Create `src/memory/decay.ts`:
- Import `TYPE_WEIGHTS`, `DEFAULT_HALF_LIFE_DAYS`, `MIN_RELEVANCE_THRESHOLD`, `MAX_OBSERVATIONS_PER_PROJECT` from `"./constants"` and `ObservationType` from `"./types"`
- `computeRelevanceScore(lastAccessed: string, accessCount: number, type: ObservationType, halfLifeDays: number = DEFAULT_HALF_LIFE_DAYS): number`:
  ```
  const ageMs = Date.now() - new Date(lastAccessed).getTime();
  const ageDays = ageMs / (24 * 60 * 60 * 1000);
  const timeDecay = Math.exp(-ageDays / halfLifeDays);
  const frequencyWeight = Math.max(Math.log2(accessCount + 1), 1);
  const typeWeight = TYPE_WEIGHTS[type];
  return timeDecay * frequencyWeight * typeWeight;
  ```
- `pruneStaleObservations(projectId: string | null, db?: Database): { pruned: number }`:
  - Get all observations for project via `getObservationsByProject(projectId, MAX_OBSERVATIONS_PER_PROJECT + 1000, db)`
  - Compute relevance score for each
  - Delete observations where score < MIN_RELEVANCE_THRESHOLD
  - If remaining count > MAX_OBSERVATIONS_PER_PROJECT, delete lowest-scored until at cap
  - Return count of pruned
  - Use `deleteObservation` from repository for each deletion (not batch DELETE for safety)

Create `src/memory/capture.ts`:
- Define `MemoryCaptureDeps` interface: `{ getDb: () => Database; projectRoot: string; }`
- `createMemoryCaptureHandler(deps: MemoryCaptureDeps)` returns async function matching `(input: { readonly event: { readonly type: string; readonly [key: string]: unknown } }) => Promise<void>`
- Internal state: `currentSessionId: string | null = null`, `currentProjectKey: string | null = null`
- Event routing (using `event.type` from the OpenCode event system, NOT the observability event type):
  - `"session.created"`: Extract sessionId from `event.properties.info.id`. Compute project key via `computeProjectKey(deps.projectRoot)`. Derive project name from `basename(deps.projectRoot)`. Call `upsertProject(...)`. Store sessionId and projectKey.
  - `"session.deleted"`: Run `pruneStaleObservations(currentProjectKey, deps.getDb())` with setTimeout(fn, 0) to defer. Reset state.
  - `"session.error"`: Extract error details from event properties. Create observation with type="error", content=`${errorType}: ${message}`, summary=first 200 chars of message (truncated, D-14 fallback). Insert via repository.
  - For observability events captured in the event store, the capture handler needs to also process decision events. Check `event.properties` for decision-shaped data. If event type matches our criteria, extract and store.

- **Event filtering per Research Pitfall 4 and Open Question 3:**
  - CAPTURE: `session.error` (as error observations), decision data from session events
  - SKIP: `tool_complete`, `context_warning`, `message.updated`, `session.idle`, `session.compacted`
  - The handler also subscribes to `session.deleted` to trigger pruning

- For phase_transition events: the handler checks event properties for phase transition data. If present, create observation with type="pattern", content describing the transition.

- All observation inserts are wrapped in try/catch (best-effort per D-14, never breaks the session). Use `setTimeout(fn, 0)` to defer non-critical writes per Research anti-pattern guidance.

- **Important:** The capture handler follows the same pattern as `createObservabilityEventHandler` in `src/observability/event-handlers.ts` — it's a factory function returning an async handler. It is a pure observer: it never modifies the event or session output.

Create `src/memory/retrieval.ts`:

Define `ScoredObservation` type:
```typescript
export type ScoredObservation = Observation & { readonly relevanceScore: number };
```

`scoreAndRankObservations(observations: readonly Observation[], halfLifeDays?: number): readonly ScoredObservation[]`:
- Compute relevanceScore for each using `computeRelevanceScore(obs.lastAccessed, obs.accessCount, obs.type, halfLifeDays)`
- Sort descending by relevanceScore
- Return as ScoredObservation array

`buildMemoryContext(options: { projectName: string; lastSessionDate: string | null; observations: readonly ScoredObservation[]; preferences: readonly Preference[]; tokenBudget?: number; }): string`:
- If no observations and no preferences, return ""
- Compute charBudget = (tokenBudget ?? DEFAULT_INJECTION_BUDGET) * CHARS_PER_TOKEN
- Build markdown string within charBudget using this structure:

```markdown
## Project Memory (auto-injected)
**Project:** {projectName}
**Last session:** {lastSessionDate || "first session"}

### Key Decisions
- {decision summaries, highest relevance first}

### Patterns
- {pattern summaries}

### Recent Errors
- {error summaries}

### Preferences
- {user preference key: value pairs}
```

- **Layer 1 (always included):** Group observations by type. For each group, include up to 5 observation summaries (one line each: `- {summary} (confidence: {confidence})`). Track char count.
- **Layer 2 (if budget allows):** After Layer 1, if charBudget remaining > 500 chars, add a "### Recent Activity" section with chronological timeline of last 5 sessions (session date + observation count). Each entry ~50 chars.
- **Layer 3 (if budget allows):** After Layer 2, if charBudget remaining > 1000 chars, expand the top 1-2 observations with full content (not just summary).
- Track `totalChars` and stop adding content when budget exhausted per D-19 and D-26.

- Each section is built as a string, measured, and only appended if it fits within remaining budget. Same approach as `buildMultiSkillContext` in `src/skills/adaptive-injector.ts`.

- Preferences section uses user-level preferences from the `preferences` table. Format: `- **{key}:** {value}`

`retrieveMemoryContext(projectPath: string, tokenBudget?: number, db?: Database): string`:
- Convenience function that ties it all together:
  1. Compute project key via `computeProjectKey(projectPath)`
  2. Get project via `getProjectByPath(projectPath, db)` — if null, return ""
  3. Get observations via `getObservationsByProject(projectKey, 100, db)`
  4. Score and rank via `scoreAndRankObservations`
  5. Get preferences via `getAllPreferences(db)`
  6. Build and return via `buildMemoryContext`

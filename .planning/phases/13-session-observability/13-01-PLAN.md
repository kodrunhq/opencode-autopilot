    **Create src/observability/types.ts** with event type definitions:
    - SessionEventType: "fallback" | "error" | "decision" | "model_switch"
    - BaseEvent: { timestamp, sessionId, type }
    - FallbackEvent: extends BaseEvent with failedModel, nextModel, reason, success
    - ErrorEvent: extends BaseEvent with errorType (from ErrorType), model, message, statusCode
    - DecisionEvent: extends BaseEvent with phase, agent, decision, rationale
    - ModelSwitchEvent: extends BaseEvent with fromModel, toModel, trigger ("fallback" | "config" | "user")
    - SessionEvent: union of all four event types

    **Create src/observability/schemas.ts** with Zod schemas:
    - baseEventSchema, fallbackEventSchema, errorEventSchema, decisionEventSchema, modelSwitchEventSchema
    - sessionEventSchema (discriminated union on type field)
    - loggingConfigSchema with retentionDays (default 30) and enabled (default true)
    - Pre-compute loggingDefaults

    **Create src/observability/session-logger.ts** with SessionLogger:
    - getLogsDir(): returns join(getGlobalConfigDir(), "logs")
    - getSessionLogPath(sessionId): returns join(getLogsDir(), `${sessionId}.jsonl`)
    - logEvent(event: SessionEvent): validate with Zod, append JSON line to session log file
    - getSessionLog(sessionId): read JSONL file, parse each line, return SessionEvent[]
    - Uses ensureDir for logs directory creation
    - Uses atomic file append (appendFile) for thread-safe writes

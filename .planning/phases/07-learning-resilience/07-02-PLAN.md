    **src/orchestrator/lesson-injection.ts** (per D-03):
    Create a new module for lesson injection utilities.

    ```typescript
    import type { Phase } from "./types";
    import type { Lesson, LessonDomain } from "./lesson-types";
    import { sanitizeTemplateContent } from "../review/sanitize";
    ```

    Export `PHASE_LESSON_DOMAINS: Readonly<Record<Phase, readonly LessonDomain[]>>`:
    - RECON: ["planning"]
    - CHALLENGE: ["architecture", "planning"]
    - ARCHITECT: ["architecture"]
    - EXPLORE: [] (no injection)
    - PLAN: ["planning", "architecture"]
    - BUILD: ["testing", "review"]
    - SHIP: ["planning"]
    - RETROSPECTIVE: [] (no injection)

    Export `buildLessonContext(lessons: readonly Lesson[], phase: Phase): string`:
    1. Get domains from `PHASE_LESSON_DOMAINS[phase]`
    2. If domains is empty, return ""
    3. Filter lessons where `domains.includes(lesson.domain)`
    4. Sort by extractedAt descending (newest first)
    5. Take first 10 (per Pitfall 5 from RESEARCH.md)
    6. Truncate each lesson.content to 256 chars (add "..." if truncated)
    7. Sanitize each content via `sanitizeTemplateContent()` (per project constraint)
    8. Format: `\n\nPrior lessons from previous runs:\n- [domain] content\n- [domain] content`
    9. Return the formatted string, or "" if no relevant lessons after filtering

    **src/tools/orchestrate.ts** (per D-03):
    Modify `processHandlerResult` to inject lessons into dispatch prompts.

    In the `case "dispatch"` block, BEFORE returning `JSON.stringify(handlerResult)`:
    1. Import `loadLessonMemory` from "../orchestrator/lesson-memory"
    2. Import `buildLessonContext` from "../orchestrator/lesson-injection"
    3. If `handlerResult.prompt` and `handlerResult.phase`:
       a. Compute `projectRoot = join(artifactDir, "..")`
       b. Load lessons: `const memory = await loadLessonMemory(projectRoot)`
       c. If memory has lessons: `const lessonCtx = buildLessonContext(memory.lessons, handlerResult.phase as Phase)`
       d. If lessonCtx is non-empty, create a new result with appended prompt: `{ ...handlerResult, prompt: handlerResult.prompt + lessonCtx }`
    4. Wrap in try/catch -- lesson injection failure should NOT break the dispatch (best-effort)
    5. Also apply the same logic in the `case "dispatch_multi"` block for each agent's prompt

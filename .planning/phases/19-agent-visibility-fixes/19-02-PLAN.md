**Step 3: Add a helper function** to build config-hook agent entries:

```typescript
function configHookAgentToEntry(agent: ConfigHookAgent): AssetEntry {
  const config = agent.config;
  return {
    name: agent.name,
    type: "agent",
    origin: "config-hook",
    mode: (config.mode as AssetEntry["mode"]) ?? undefined,
    group: agent.group,
    hidden: (config.hidden as boolean) ?? false,
  };
}
```

Note: Do NOT include `model` field resolution in stocktakeCore itself. Model resolution requires plugin config state (groups, overrides). Instead, the model field remains undefined for config-hook agents in the core function. The tool wrapper (`ocStocktake`) can optionally enrich with model data later. This keeps `stocktakeCore` as a pure function per the existing pattern.

**Step 5: After the filesystem agent scan loop** (after line ~126), add config-hook agent injection:

```typescript
// Add config-hook agents (skip any already found on filesystem to avoid duplicates)
const filesystemAgentNames = new Set(agents.map((a) => a.name));
if (configHookAgents) {
  for (const hookAgent of configHookAgents) {
    if (!filesystemAgentNames.has(hookAgent.name)) {
      agents.push(configHookAgentToEntry(hookAgent));
    }
  }
}
```

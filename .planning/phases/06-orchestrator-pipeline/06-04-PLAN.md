    **State management in orchestrateCore:**
    - After handler returns `complete`: call `completePhase(state)` + `saveState()`, then invoke NEXT phase handler
    - After handler returns `dispatch`/`dispatch_multi`: if handler included state updates, apply them with `patchState()` + `saveState()`
    - Never mutate state directly (per D-17)

    Remove the `PHASE_AGENTS` constant (replaced by `AGENT_NAMES` in handlers/types.ts and `PHASE_HANDLERS` in handlers/index.ts).

    **Tool** (`src/tools/review.ts`) -- per D-08, D-09, D-16, REVW-01:

    Follow the `*Core` + `tool()` wrapper pattern from `src/tools/orchestrate.ts`.

    ```typescript
    import { tool } from "@opencode-ai/plugin";
    ```

    **reviewCore args interface:**
    ```typescript
    interface ReviewArgs {
      readonly scope?: string;
      readonly filter?: string;
      readonly directory?: string;
      readonly findings?: string;
    }
    ```

    Export `reviewCore(args: ReviewArgs, projectRoot: string): Promise<string>`:

    The tool is stateful between invocations. Pipeline state stored at `{projectRoot}/.opencode-assets/current-review.json`.

    **Logic:**
    1. Try to load existing review state from `current-review.json`
    2. If NO state AND scope provided: Start new review
       - Call `startReview(scope, projectRoot, { filter, directory })`
       - Save state to `current-review.json` (atomic write)
       - Load memory, inject `{{MEMORY}}` with false positive context into dispatch prompts
       - Return JSON: `{ action: "dispatch", stage: 1, agents: [...] }`
    3. If NO state AND NO scope: Error
       - Return JSON: `{ action: "error", message: "No active review. Provide scope to start." }`
    4. If state EXISTS AND findings provided: Advance pipeline
       - Call `advancePipeline(findings, currentState)`
       - If result.action === "dispatch": save updated state, return dispatch instructions
       - If result.action === "complete":
         a. Build report, update memory with findings, save memory
         b. Delete `current-review.json` state file
         c. Return JSON with complete report
    5. If state EXISTS AND NO findings: Return current status
       - Return JSON: `{ action: "status", stage: currentState.stage, message: "Awaiting findings from dispatched agents" }`

    **State persistence:**
    - File: `{artifactDir}/current-review.json`
    - Validated through `reviewStateSchema`
    - Atomic writes (tmp + rename)
    - Cleared on completion or error

    **Memory integration:**
    - On start: load memory, include false positives in agent prompts via `{{MEMORY}}`
    - On complete: update memory with new findings (append to recentFindings), update projectProfile stacks, save

    **Error handling:**
    - Wrap everything in try/catch
    - Return `{ action: "error", message: "..." }` on any failure
    - Never throw from reviewCore (tool must always return JSON)

    Export `ocReview = tool({...})`:
    - description: "Run multi-agent code review. Provide scope (staged|unstaged|branch|all|directory) to start, or findings from dispatched agents to advance the pipeline. Returns JSON with action (dispatch|complete|error)."
    - args:
      - scope: tool.schema.enum(["staged", "unstaged", "branch", "all", "directory"]).optional()
      - filter: tool.schema.string().optional().describe("Regex pattern to filter files")
      - directory: tool.schema.string().optional().describe("Directory path for directory scope")
      - findings: tool.schema.string().optional().describe("JSON findings from previously dispatched review agents")
    - execute: calls `reviewCore(args, process.cwd())`

    **Plugin registration** (update `src/index.ts`):
    - Add import: `import { ocReview } from "./tools/review";`
    - Add to tool object: `oc_review: ocReview`

    - Test uses temp directory as projectRoot to avoid polluting real filesystem

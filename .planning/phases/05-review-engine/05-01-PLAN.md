    - getDiffAnalysis categorizes changed files into categories (auth, test, api, config, etc.)
    - File filter pattern correctly filters returned file list
  </behavior>
  <action>
    **Stack Detection** (`src/review/stack-detection.ts`) -- per D-04, D-05:

    Define `STACK_MARKERS` as a frozen readonly array of `{ file: string, stack: string }` objects:
    - package.json -> node
    - tsconfig.json -> typescript
    - Cargo.toml -> rust
    - go.mod -> go
    - pyproject.toml -> python
    - requirements.txt -> python
    - Gemfile -> ruby
    - pom.xml -> java
    - build.gradle -> java
    - composer.json -> php

    Export `ProjectStack` interface: `{ readonly detectedStacks: readonly string[]; readonly markerFiles: readonly string[] }`

    Export `detectProjectStack(projectRoot: string): Promise<ProjectStack>`:
    - Use `Promise.all` with `access(join(projectRoot, file))` for each marker
    - Deduplicate stacks (python has 2 markers)
    - Return `Object.freeze({ detectedStacks, markerFiles })`
    - Root level only -- join projectRoot directly with marker file name, no recursion

    **Diff Resolver** (`src/review/diff-resolver.ts`) -- per D-09:

    Export `ReviewScope` type = "staged" | "unstaged" | "branch" | "all" | "directory"

    Export `DiffResult` interface: `{ readonly scope: ReviewScope; readonly diff: string; readonly files: readonly string[] }`

    Export `DiffAnalysis` interface: `{ readonly categories: readonly string[]; readonly hasTests: boolean; readonly hasAuth: boolean; readonly hasConfig: boolean; readonly fileCount: number }`

    Export `getDiff(scope, projectRoot, options?)`: Promise<DiffResult>:
    - Use `promisify(execFile)` from node:child_process
    - Always include `--no-color --no-ext-diff` flags (Pitfall 3)
    - For "staged": `git diff --cached`
    - For "unstaged": `git diff`
    - For "branch": `git diff main...HEAD`
    - For "all": `git diff HEAD`
    - For "directory": `git diff --cached -- {directory}`
    - Get file list separately with `--name-only`
    - Apply optional filter regex to file list
    - maxBuffer: 10 * 1024 * 1024 for diff, 1 * 1024 * 1024 for names
    - Return Object.freeze result

    Export `getDiffAnalysis(files: readonly string[]): DiffAnalysis`:
    - Pure function, no I/O
    - Categorize files: auth (path includes auth/), test (*.test.*, *.spec.*), api (api/, routes/), config (.env*, *.config.*)
    - Return frozen object

    **Tests**: Use temp directories for stack detection. For diff resolver, test `getDiffAnalysis` as a pure function. For `getDiff`, test the argument construction logic (mock execFile or test in a real git repo).
  </action>

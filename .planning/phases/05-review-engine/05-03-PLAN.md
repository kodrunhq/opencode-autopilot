  <action>
    **Pipeline** (`src/review/pipeline.ts`) -- per D-06:

    Export `ReviewStageResult` interface:
    ```typescript
    export interface ReviewStageResult {
      readonly action: "dispatch" | "complete" | "error";
      readonly stage?: number;
      readonly agents?: readonly { readonly name: string; readonly prompt: string }[];
      readonly report?: ReviewReport;
      readonly message?: string;
      readonly state?: ReviewState;  // Serialized state for persistence
    }
    ```

    Export `startReview(scope: ReviewScope, projectRoot: string, options?: { filter?: string; directory?: string }): Promise<ReviewStageResult>`:
    1. Call `detectProjectStack(projectRoot)`
    2. Call `getDiff(scope, projectRoot, options)`
    3. Call `getDiffAnalysis(diff.files)`
    4. Call `selectAgents(stack, analysis, REVIEW_AGENTS)`
    5. If diff is empty, return `{ action: "complete", report: emptyReport(scope) }`
    6. Build dispatch instructions: for each selected agent, replace `{{DIFF}}` in prompt with actual diff, clear `{{PRIOR_FINDINGS}}` and `{{MEMORY}}` placeholders
    7. Create initial ReviewState: `{ stage: 1, selectedAgents: [...], accumulatedFindings: [], scope, startedAt: new Date().toISOString() }`
    8. Return `{ action: "dispatch", stage: 1, agents: [...], state }`

    Export `advancePipeline(findingsJson: string, currentState: ReviewState): ReviewStageResult`:
    1. Parse findings from JSON string (robust extraction per Pitfall 6: look for `[{` or `{"findings"`, try JSON.parse, validate each through reviewFindingSchema, discard unparseable)
    2. Accumulate with prior findings: `[...currentState.accumulatedFindings, ...parsedFindings]`
    3. Based on `currentState.stage`:
       - **Stage 1 -> 2:** Build cross-verification prompts via `buildCrossVerificationPrompts`. Return dispatch.
       - **Stage 2 -> 3:** Build red-team + product-thinker prompts (from STAGE3_AGENTS). Include ALL accumulated findings as {{PRIOR_FINDINGS}}. Return dispatch.
       - **Stage 3 -> 4:** Check if any CRITICAL/HIGH findings have actionable suggestions. If yes, return fix instructions. If no, build final report and return complete.
       - **Stage 4 -> complete:** Build final report with all findings including re-verify results. Return complete.
    4. Update state with new stage number and accumulated findings.

    Export `parseAgentFindings(raw: string, agentName: string): readonly ReviewFinding[]`:
    - Extract JSON from raw text (may contain markdown, code blocks)
    - Look for `{"findings": [...]}` or raw array `[{...}]`
    - Validate each finding through reviewFindingSchema.safeParse
    - Set agent field to agentName if missing
    - Return valid findings, discard invalid with warning

    **Report** (`src/review/report.ts`) -- per D-11, REVW-10:

    Export `buildReport(findings: readonly ReviewFinding[], scope: string, agentsRan: readonly string[]): ReviewReport`:
    1. Deduplicate findings
    2. Sort: group by file, within each file sort by severity (CRITICAL > HIGH > MEDIUM > LOW)
    3. Compute summary counts: CRITICAL, HIGH, MEDIUM, LOW, total
    4. Validate through reviewReportSchema.parse
    5. Return frozen report

    Export `deduplicateFindings(findings: readonly ReviewFinding[]): readonly ReviewFinding[]`:
    - Key: `${agent}:${file}:${line}:${severity}`
    - On collision: keep higher severity version
    - Return new array (no mutation)

    Severity order for sorting: `const SEVERITY_ORDER = { CRITICAL: 0, HIGH: 1, MEDIUM: 2, LOW: 3 } as const`

    - buildReport computes correct summary counts
    - deduplicateFindings removes same-key duplicates, keeps higher severity
    - startReview returns empty report for empty diff (use mock/stub for git commands)
    - advancePipeline transitions stage correctly

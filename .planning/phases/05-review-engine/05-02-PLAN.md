    **Prompt design rules (from research):**
    - Target 500-800 tokens per prompt BEFORE diff injection
    - Structure: Role statement -> Instructions (what to check) -> Output format (JSON finding) -> Placeholders
    - Include `{{DIFF}}` placeholder for diff content
    - Include `{{PRIOR_FINDINGS}}` placeholder for cross-verification (Stage 2)
    - Include `{{MEMORY}}` placeholder for false positive suppression
    - Remove ace's "Hard Gates" framing -- enforce via TypeScript logic
    - Remove tool permission instructions -- agents don't use tools in this architecture
    - Keep the specific checks/rules that define each agent's expertise

    **Per agent (port logic from ace source, compress to essential behavioral contract):**

    1. `logic-auditor.ts` -- logicAuditor: Trace every changed function, check loops/boundaries/null/async, off-by-one errors, boundary correctness (< vs <=), null/undefined on property access, async/await correctness, unreachable code. severityFocus: CRITICAL, HIGH.

    2. `security-auditor.ts` -- securityAuditor: Hardcoded secrets (API keys, passwords, tokens in code), input validation gaps, SQL injection (non-parameterized queries), XSS (unsanitized HTML), CSRF, auth/authz bypass, rate limiting gaps, error messages leaking internals. severityFocus: CRITICAL, HIGH.

    3. `code-quality-auditor.ts` -- codeQualityAuditor: Function length (>50 lines), file size (>800 lines), nesting depth (>4 levels), duplication, naming consistency, dead code, magic numbers, missing error handling. severityFocus: MEDIUM, LOW.

    4. `test-interrogator.ts` -- testInterrogator: Assertion quality (no `expect(true).toBe(true)`), tautological tests, over-mocking (mock returns what test expects), behavioral coverage (changed code has corresponding test), missing edge case tests. severityFocus: HIGH, MEDIUM.

    5. `silent-failure-hunter.ts` -- silentFailureHunter: Empty catch blocks, catch-log-only (catch + console.log but no throw/return), optional chaining masking errors, generic catch-all handlers, missing finally cleanup, async errors that silently swallow. severityFocus: HIGH, MEDIUM.

    6. `contract-verifier.ts` -- contractVerifier: Both sides of every API boundary must agree on shape/method/URL, request/response type mismatches, missing fields consumed downstream, version mismatches in API contracts, broken imports. severityFocus: CRITICAL, HIGH.

    **Output format instruction in each prompt:**
    ```
    For each finding, output a JSON object:
    {"file": "path/to/file", "line": 42, "severity": "HIGH", "agent": "{agent-name}", "finding": "description", "suggestion": "how to fix"}

    If no findings: {"findings": []}
    Wrap all findings in: {"findings": [...]}
    ```

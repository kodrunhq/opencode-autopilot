<objective>
Implement the three creation tools (oc_create_agent, oc_create_skill, oc_create_command), register them in the plugin entry point, and create the slash command markdown files.

Purpose: This is the core deliverable of Phase 2 — the actual tools that let users scaffold new OpenCode extensions from within a session. Each tool validates input, generates markdown via templates (from Plan 01), writes to ~/.config/opencode/, and returns a success message with restart instructions.

Output: Three tool modules, updated index.ts, three command markdown files, and integration tests.
</objective>

    - Uses `import { tool } from "@opencode-ai/plugin"` for tool factory
    - Defines Zod schema via `tool.schema` (per D-12, all params in single call)
    - Validates name via validators (per D-04, D-05, D-09)
    - Checks file existence via `fileExists()` (per D-13, never overwrite)

    **src/tools/create-agent.ts** — `ocCreateAgent`:
    - Zod args (per D-01, D-02, D-12):
      - `name`: `tool.schema.string().min(1).max(64).describe("Agent name (e.g., 'code-reviewer')")`
      - `description`: `tool.schema.string().min(1).max(500).describe("What the agent does")`
      - `mode`: `tool.schema.enum(["primary", "subagent", "all"]).default("subagent").describe("Agent mode")`
      - `model`: `tool.schema.string().optional().describe("Model identifier (omit for model-agnostic)")`
      - `temperature`: `tool.schema.number().min(0).max(1).optional().describe("Temperature 0.0-1.0")`
    - Execute flow: validate name -> check file exists -> generate markdown -> ensureDir -> writeFile -> return success
    - Target path: `join(getGlobalConfigDir(), "agents", `${name}.md`)` (per D-10, always global)

    **src/tools/create-skill.ts** — `ocCreateSkill`:
    - Zod args (per D-05, D-06):
      - `name`: `tool.schema.string().min(1).max(64).describe("Skill name (1-64 chars, lowercase with hyphens)")`
      - `description`: `tool.schema.string().min(1).max(1024).describe("What the skill provides")`
      - `license`: `tool.schema.string().optional().describe("License (e.g., 'MIT')")`
      - `compatibility`: `tool.schema.string().optional().describe("Compatibility (e.g., 'opencode')")`
    - Execute flow: validate name -> check SKILL.md exists -> generate markdown -> ensureDir (creates skill directory) -> writeFile SKILL.md -> return success
    - Target path: `join(getGlobalConfigDir(), "skills", name, "SKILL.md")` (per D-07, directory matches name)
    - File to check: the SKILL.md path (per D-13)

    **src/tools/create-command.ts** — `ocCreateCommand`:
    - Zod args (per D-08):
      - `name`: `tool.schema.string().min(1).max(64).describe("Command name (e.g., 'review')")`
      - `description`: `tool.schema.string().min(1).max(500).describe("What the command does")`
      - `agent`: `tool.schema.string().optional().describe("Agent to use (e.g., 'code-reviewer')")`
      - `model`: `tool.schema.string().optional().describe("Model override")`
    - Execute flow: validate command name (includes built-in check) -> check file exists -> generate markdown -> ensureDir -> writeFile -> return success
    - Target path: `join(getGlobalConfigDir(), "commands", `${name}.md`)` (per D-10)

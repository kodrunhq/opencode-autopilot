# Phase 12 Plan 01: Fix Zen Model Provider Prefix in Configure Wizard

## Objective

Fix the configure wizard (both tool-based `oc_configure` and interactive CLI `configure-tui`) to show the full provider-prefixed model IDs that match OpenCode's native `/models` display, ensuring users can distinguish Go vs Zen providers.

## Context

- @src/tools/configure.ts - Tool-based configure with SDK provider data
- @bin/configure-tui.ts - Interactive CLI configure with `opencode models` output
- @tests/tools/configure.test.ts - Existing test coverage
- @12-CONTEXT.md - Phase context with decisions D-01 through D-06

## Tasks

<task id="1" type="auto">
### Fix model ID construction in configure.ts

The `discoverAvailableModels()` function constructs model IDs as `${provider.id}/${modelKey}` where `modelKey` comes from `Object.keys(provider.models)`. This can lose the Zen prefix if the model key doesn't include it. The model objects have an `id` field that should be the canonical identifier. Fix the function to use the model's `id` field when it already contains a `/` (indicating it has a provider prefix), and fall back to `${provider.id}/${modelKey}` otherwise.

Also update `buildNumberedModelList` to show the model's friendly name (from the `name` field) alongside the ID for better UX.

**Done criteria:**
- `discoverAvailableModels` uses model `id` field when it contains a provider prefix
- Models display with correct provider prefixes matching OpenCode's native output
- Existing tests updated and all pass
</task>

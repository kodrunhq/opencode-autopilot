Create the directory `.github/workflows/` if it does not exist, then create `.github/workflows/ci.yml` with this exact content:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: "1.3.11"

      - run: bun install --frozen-lockfile

      - name: Lint and format check
        run: bun run lint

      - name: Type check
        run: bunx tsc --noEmit

      - name: Test with coverage
        run: bun test --coverage --bail 1
```

Design decisions (from RESEARCH.md):
- **Single job:** The entire check runs in under 30 seconds total. Splitting into parallel jobs adds 3x setup overhead for zero benefit.
- **`--frozen-lockfile`:** Prevents CI from silently upgrading dependencies.
- **`--bail 1`:** Fast-fail on first test failure to save CI minutes.
- **Bun pinned to 1.3.11:** Matches local dev environment, prevents version drift (Pitfall 3 from research).
- **No coverage upload:** At 97% coverage with bunfig.toml thresholds, Codecov adds complexity without value.
- **ubuntu-latest only:** Start single-platform. Add matrix later if cross-OS issues surface.
- **Step order:** lint (fastest, catches formatting) -> type-check (catches type errors) -> test (slowest, catches behavioral regressions). Fail-fast ordering.

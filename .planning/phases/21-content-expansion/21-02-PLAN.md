**1a. Create assets/commands/oc-review-agents.md (per D-12, D-13):**

Create a prompt-only command following the oc-review-pr pattern. Frontmatter:
```yaml
---
description: Review and improve your project's agents.md file with structure validation and prompt quality feedback
argument-hint: "[path-to-agents.md]"
---
```

No `agent:` field — this command runs in the current agent context (like oc-stocktake).

Command body instructs the LLM to:

1. **Locate the agents.md file:**
   - If `$ARGUMENTS` provided, use that path
   - Otherwise, check `.opencode/agents.md`, then `agents.md` in project root
   - If not found, suggest creating one and offer to show starter templates

2. **Read and parse the file:**
   - Parse each agent block (look for `## Agent:` or similar headers)
   - Extract: name, description, system prompt, tool permissions, model assignment

3. **Validate structure** (score 0-3 per agent):
   - Has a clear name and description (+1)
   - Has a system prompt with specific instructions, not generic (+1)
   - Has explicit tool permissions (allow/deny lists) (+1)
   - Deduct 1 for: missing description, empty system prompt, no tool configuration

4. **Assess prompt quality** (score 0-4 per agent):
   - Role clarity: Does the prompt define WHO the agent is? (+1)
   - Task specificity: Does the prompt define WHAT the agent does with concrete instructions? (+1)
   - Guardrails: Does the prompt define what NOT to do or boundaries? (+1)
   - Output format: Does the prompt specify HOW to format responses? (+1)

5. **Check coverage** for project type:
   - Read project manifest files to detect type (package.json, pom.xml, etc.)
   - Compare agents against recommended agents for that project type
   - Note missing agent roles that would benefit the project
   - Reference starter templates in `~/.config/opencode/templates/` for comparison

6. **Output the review:**

```
## agents.md Review

**File:** {path}
**Agents found:** {count}
**Overall Score:** {total}/{max} ({percentage}%)

### Per-Agent Assessment

#### {agent-name}
- Structure: {score}/3 — {brief notes}
- Prompt Quality: {score}/4 — {brief notes}
- Suggestions:
  - {specific improvement}

### Coverage Analysis
- Project type detected: {type}
- Recommended agents: {list}
- Missing: {list with brief explanation of why each would help}
- Starter template: Run `cat ~/.config/opencode/templates/{type}.md` to see a reference

### Top 3 Improvements
1. {Most impactful improvement}
2. {Second most impactful}
3. {Third most impactful}
```

**1b. Create assets/templates/ directory and four starter templates (per D-14, D-15):**

Each template is a curated agents.md file that a user can copy directly into their project. Templates use OpenCode's agents.md format (not YAML frontmatter — these are sample `agents.md` files for user projects).

**assets/templates/web-api.md:**
A starter agents.md for REST API / backend projects. Include 4-5 agents:
- **api-designer**: Designs RESTful endpoints, validates OpenAPI contracts, ensures consistent response formats
- **db-architect**: Reviews schema design, migration safety, query optimization, index strategy
- **security-auditor**: Checks auth flows, input validation, injection prevention, CORS configuration
- **test-engineer**: Writes integration tests for API endpoints, mocks external services, validates error responses
- **devops**: Reviews Dockerfiles, CI pipelines, environment configuration, deployment readiness

Each agent has: description, a 3-5 sentence system prompt with specific instructions, and tool permissions.

**assets/templates/cli-tool.md:**
A starter agents.md for CLI tool projects. Include 4 agents:
- **ux-writer**: Reviews help text, error messages, command naming, flag conventions
- **arg-parser-expert**: Validates argument parsing, subcommand structure, completions
- **test-engineer**: Writes unit tests for commands, integration tests for workflows, snapshot tests for output
- **release-manager**: Reviews changelogs, version bumping, distribution packaging

**assets/templates/library.md:**
A starter agents.md for reusable library projects. Include 4 agents:
- **api-designer**: Reviews public API surface, naming conventions, backward compatibility
- **docs-writer**: Generates README sections, API documentation, usage examples, migration guides
- **test-engineer**: Writes unit tests with edge cases, property-based tests, compatibility tests
- **perf-analyst**: Reviews bundle size, benchmarks, memory usage, tree-shaking compatibility

**assets/templates/fullstack.md:**
A starter agents.md for fullstack web apps. Include 5-6 agents:
- **frontend-architect**: Reviews component structure, state management, routing, responsive design
- **backend-architect**: Reviews API design, database schema, authentication, authorization
- **security-auditor**: Full-stack security review (XSS, CSRF, auth, secrets management)
- **test-engineer**: E2E tests for critical flows, API integration tests, component unit tests
- **ux-reviewer**: Reviews accessibility, loading states, error states, mobile experience
- **devops**: Reviews deployment config, environment management, monitoring setup

Each template should be immediately usable — a user copies it as their `.opencode/agents.md` and has a working setup. Include a header comment explaining the template purpose and how to customize.

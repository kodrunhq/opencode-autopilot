- **Availability Test:** How to verify the agent appears (Tab cycle for primary agents, @ autocomplete for subagents)
- **Skill Loading Test:** Which skills should be loaded (read from agent definitions)
- **Core Behavior Test:** One representative task exercising the agent's purpose
- **Negative Test:** One scenario (e.g., invoking agent without required context)
- **Pass/Fail:** Clear criteria

Note: Primary agents (autopilot, debugger, planner, reviewer) should test Tab-cycle visibility. Subagents (researcher, metaprompter, documenter, pr-reviewer) should test @-mention accessibility.

CID Alphabet and hash spec (from RESEARCH.md):
```typescript
const CID_ALPHABET = "ZPMQVRWSNKTXJBYH"; // 16 chars, omo's alphabet

function fnv1a(str: string): number {
  let hash = 0x811c9dc5; // FNV offset basis
  for (let i = 0; i < str.length; i++) {
    hash ^= str.charCodeAt(i);
    hash = Math.imul(hash, 0x01000193); // FNV prime
  }
  return hash >>> 0;
}

function computeLineHash(content: string): string {
  const h = fnv1a(content);
  return CID_ALPHABET[h & 0xf] + CID_ALPHABET[(h >> 4) & 0xf];
}
```

LINE#ID format: "42#VK" where 42 is 1-based line number, VK is 2-char CID hash.
Three ops: replace (single line or range), append (after anchor), prepend (before anchor).

    interface HashlineEdit {
      readonly op: "replace" | "append" | "prepend";
      readonly pos: string;      // "LINE#HASH" anchor
      readonly end?: string;     // End anchor for range replace
      readonly lines: string | readonly string[] | null;  // null = delete
    }
    interface HashlineEditArgs {
      readonly file: string;
      readonly edits: readonly HashlineEdit[];
    }
    ```

    **Core function: `hashlineEditCore(args: HashlineEditArgs): Promise<string>`**
    1. Read file at `args.file` using `readFile` from `node:fs/promises`
    2. Split into lines array (preserve empty trailing line if present)
    3. Sort edits by line number descending (bottom-up) to prevent drift (D-09)
    4. For each edit:
       a. Parse `pos` anchor — extract line number and hash
       b. Validate line number is within file bounds
       c. Compute hash of current line content at that position
       d. Compare computed hash vs provided hash
       e. If mismatch: collect error with updated LINE#ID for that line and 2 surrounding lines above/below
       f. If match: apply operation:
          - `replace` (no `end`): Replace single line. `lines` = string replaces the line, `lines` = string[] replaces with multiple lines, `null` deletes the line
          - `replace` (with `end`): Parse end anchor, validate, replace range from pos to end (inclusive). Same lines handling.
          - `append`: Insert `lines` content after the anchor line
          - `prepend`: Insert `lines` content before the anchor line
    5. If any validation errors: return error message with all mismatches and updated anchors
    6. Write modified content back to file using `writeFile` from `node:fs/promises` (regular write, NOT wx flag — this is an edit, not a create)
    7. Return success message with count of edits applied

    **Tool wrapper: `ocHashlineEdit`**

    export const ocHashlineEdit = tool({
      description: "Edit files using hash-anchored line references (LINE#ID format). Validates line content hasn't changed before applying edits. Supports replace, append, and prepend operations.",
      args: {
        file: tool.schema.string().describe("Absolute path to the file to edit"),
        edits: tool.schema.array(
          tool.schema.object({

            pos: tool.schema.string().describe("LINE#HASH anchor, e.g. '42#VK'"),
            end: tool.schema.string().optional().describe("End anchor for range replace, e.g. '48#SN'"),
            lines: tool.schema.union([
              tool.schema.string(),
              tool.schema.array(tool.schema.string()),
              tool.schema.null(),
            ]).describe("New content (string, string[], or null to delete)"),
          })
        ).describe("Array of edit operations to apply"),
      },
      async execute(args) {
        return hashlineEditCore(args);
      },
    });
    ```

    Define frozen constants:

    `CODE_EXTENSIONS` -- frozen Set of code file extensions per D-14:
    ```typescript
    export const CODE_EXTENSIONS: ReadonlySet<string> = Object.freeze(new Set([
      ".ts", ".tsx", ".js", ".jsx", ".py", ".go", ".rs", ".java", ".cs", ".rb", ".cpp", ".c", ".h",
    ]));
    ```

    `EXT_COMMENT_STYLE` -- maps file extension to comment prefix:
    ```typescript
    export const EXT_COMMENT_STYLE: Readonly<Record<string, string>> = Object.freeze({
      ".ts": "//", ".tsx": "//", ".js": "//", ".jsx": "//",
      ".java": "//", ".cs": "//", ".go": "//", ".rs": "//",
      ".c": "//", ".cpp": "//", ".h": "//",
      ".py": "#", ".rb": "#",
    });
    ```

    `COMMENT_PATTERNS` -- regex to extract comment text from a line:
    ```typescript
    export const COMMENT_PATTERNS: Readonly<Record<string, RegExp>> = Object.freeze({
      "//": /^\s*\/\/\s*(.+)/,
      "#": /^\s*#\s*(.+)/,
    });
    ```

    `SLOP_PATTERNS` -- frozen array of regex patterns matching against extracted comment text only (per D-13):
    ```typescript
    export const SLOP_PATTERNS: readonly RegExp[] = Object.freeze([
      /^increment\s+.*\s+by\s+\d+$/i,
      /^decrement\s+.*\s+by\s+\d+$/i,
      /^set\s+\w+\s+to\s+/i,
      /^return\s+the\s+(result|value|data)/i,
      /^(?:this|the)\s+(?:function|method|class)\s+(?:does|will|is used to|handles)/i,
      /^(?:initialize|init)\s+(?:the\s+)?(?:variable|value|state)/i,
      /^loop\s+(?:through|over)\s+/i,
      /^check\s+if\s+/i,
      /^import\s+(?:the\s+)?(?:necessary|required|needed)/i,
      /^define\s+(?:the\s+)?(?:interface|type|class|function)/i,
      /\belegantly?\b/i,
      /\brobust(?:ly|ness)?\b/i,
      /\bcomprehensive(?:ly)?\b/i,
      /\bseamless(?:ly)?\b/i,
      /\blever(?:age|aging)\b/i,
      /\bpowerful\b/i,
      /\bsophisticated\b/i,
      /\bstate[\s-]of[\s-]the[\s-]art\b/i,
      /\bcutting[\s-]edge\b/i,
    ]);
    ```

    **2. Create `src/hooks/anti-slop.ts`:**

    Export `SlopFinding` interface with `readonly line: number`, `readonly text: string`, `readonly pattern: string`.

    Export `isCodeFile(filePath: string): boolean` -- checks if extname is in CODE_EXTENSIONS.

    Export `scanForSlopComments(content: string, ext: string): readonly SlopFinding[]`:
    - Look up comment style from EXT_COMMENT_STYLE[ext]
    - Look up comment regex from COMMENT_PATTERNS[commentStyle]
    - Split content by newlines, for each line attempt comment regex match
    - If match, extract comment text (group 1), trim, test against each SLOP_PATTERNS
    - On first pattern match, push a frozen SlopFinding with line number (1-based), text, pattern.source
    - Return frozen array

    Export `createAntiSlopHandler(options: { showToast: (title: string, message: string, variant: string) => Promise<void> })`:
    - Returns an async function matching the tool.execute.after signature
    - Define `FILE_WRITING_TOOLS` as a Set: `["write_file", "edit_file", "write", "edit", "create_file"]`
    - Guard: if tool not in set, return early
    - Extract file path from hookInput.args by checking keys "file_path", "filePath", "path", "file"
    - Guard: if no path or not isCodeFile, return early
    - Use output.output as the content to scan (per Research pitfall 2: avoids stale file reads)
    - Call scanForSlopComments(content, ext)
    - If findings non-empty, call showToast with "Anti-Slop Warning", formatted findings (first 5 lines), "warning"
    - Wrap showToast in try/catch (best-effort per plugin patterns)

    Per D-12: warn only, non-blocking. Per D-13: curated regex list. Per D-14: code file types only.
